# Asset Tokenization: Financial Design of Student Accommodation (RHL_X)

## 1. Project Overview
This repository contains the technical implementation for the **IFTE0007 Individual Coursework**. The project aims to digitise the tenancy rights of Eleanor Rosa House (Room X) into a series of 12 time-sliced Non-Fungible Tokens (NFTs). This design transforms a static rental agreement into a liquid, yield-bearing financial asset.

## 2. Technical Specifications
- **Token Standard:** ERC-721 (Non-Fungible Token)
- **Royalty Standard:** ERC-2981
- **Blockchain:** Ethereum Sepolia Testnet
- **Network Architecture:** Three-tier mapping (Physical Layer ↔ Data Layer ↔ Contract Layer)

## 3. Core Functional Logic
### A. Discrete Temporal Divisibility (`MAX_SUPPLY = 12`)
The contract strictly limits the minting process to **12 units**, representing the 12-month lease period for the 2026-2027 academic year. This programmatic constraint ensures the structural scarcity of the asset and prevents unauthorized inflation of usage rights.

### B. Programmable Royalty Mechanism (`ERC-2981`)
A **2% protocol fee (200 basis points)** is hard-coded into the contract. 
- **Economic Rationale:** This mechanism ensures that the property management/landlord captures a share of the value generated by secondary market liquidity. It aligns the interests of the physical asset owner with the tokenised secondary market.

### C. Cyber-Physical Integration
The `baseURI` parameter links each NFT to an IPFS-hosted JSON file. In a production environment, this URI serves as the authenticated data source for **Smart Lock API Access**. Ownership of a specific TokenID on-chain triggers the offline generation of room access codes for the corresponding month.
